### 一、ER图

####  1.1 实体关系总览


本系统包含 **6个核心实体**，对应数据库中的6张表：

| 实体名 | 表名 | 中文名称 | 说明 |

| User | user | 用户 | 统一账号体系，存储登录信息 |
| Student | student | 学生 | 学生扩展信息，与用户一对一关联 |
| Building | building | 楼栋 | 宿舍楼栋信息 |
| Dormitory | dormitory | 宿舍 | 宿舍房间信息，属于某楼栋 |

#### 1.2 完整关系列表

| 序号 | 父实体 | 关系 | 子实体 | 外键字段 | 说明 |
|:----:|--------|:----:|--------|----------|------|
| 1 | User | **1:1** | Student | `student.user_id` | 一个学生对应一个用户账号 |
| 2 | User | **1:N** | Building | `building.building_admin_id` | 一个楼栋管理员管理多栋楼 |
| 3 | Building | **1:N** | Dormitory | `dormitory.building_id` | 一栋楼有多间宿舍 |
| 4 | Dormitory | **1:N** | Student | `student.dormitory_id` | 一间宿舍住多名学生 |
| 5 | Student | **1:N** | Repair | `repair.reporter_id` | 一个学生提交多个报修 |
| 6 | Dormitory | **1:N** | Repair | `repair.dormitory_id` | 一间宿舍有多个报修 |
| 7 | User | **1:N** | Repair | `repair.handler_id` | 一个楼管处理多个报修 |
| 8 | Student | **1:N** | CheckRecord | `check_record.student_id` | 一个学生有多条入住记录 |
| 9 | Dormitory | **1:N** | CheckRecord | `check_record.from_dormitory_id`<br>`check_record.to_dormitory_id` | 一条记录关联两个宿舍（换宿场景） |
| 10 | User | **1:N** | CheckRecord | `check_record.operator_id` | 一个操作员办理多条入住记录 |

#### 1.3 关系类型说明

| 关系类型 | 符号 | 说明 | 示例 |
|----------|:----:|------|------|
| **1:1 一对一** | 1 : 1 | 父表一条记录对应子表一条记录 | 一个用户 ↔ 一个学生 |
| **1:N 一对多** | 1 : N | 父表一条记录对应子表多条记录 | 一栋楼 ↔ 多间宿舍 |
| **N:1 多对一** | N : 1 | 子表多条记录对应父表一条记录 | 多间宿舍 ↔ 一栋楼 |

### 二、完整建表SQL脚本

```sql
-- =====================================================
-- 数据库：dormitory_system
-- 字符集：utf8mb4（支持emoji和特殊字符）
-- =====================================================

CREATE DATABASE IF NOT EXISTS dormitory_system 
DEFAULT CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

USE dormitory_system;

-- =====================================================
-- 1. 用户表（统一账号体系）
-- 功能覆盖：登录、权限控制、用户管理、操作日志关联
-- =====================================================
CREATE TABLE user (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(50) UNIQUE NOT NULL COMMENT '登录名（唯一）',
    password VARCHAR(100) NOT NULL COMMENT '加密密码（MD5+盐）',
    salt VARCHAR(20) COMMENT '密码盐值（8位随机）',
    real_name VARCHAR(50) NOT NULL COMMENT '真实姓名',
    phone VARCHAR(20) COMMENT '手机号',
    role VARCHAR(20) NOT NULL DEFAULT 'STUDENT' COMMENT '角色：ADMIN/BUILDING_ADMIN/STUDENT',
    status TINYINT DEFAULT 1 COMMENT '状态：1启用 0禁用（软删除用）',
    create_time DATETIME DEFAULT NOW() COMMENT '创建时间',
    update_time DATETIME DEFAULT NOW() ON UPDATE NOW() COMMENT '更新时间',
    last_login_time DATETIME COMMENT '最后登录时间',
    last_login_ip VARCHAR(50) COMMENT '最后登录IP',
    INDEX idx_username (username),
    INDEX idx_role (role),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户账号表';


-- =====================================================
-- 2. 学生信息表（学生特有扩展信息）
-- 功能覆盖：学生管理、入住/退宿/换宿、我的宿舍
-- =====================================================
CREATE TABLE student (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '学生记录ID',
    user_id INT UNIQUE NOT NULL COMMENT '关联user.id（一对一）',
    student_no VARCHAR(20) UNIQUE NOT NULL COMMENT '学号（唯一）',
    gender VARCHAR(10) COMMENT '性别：男/女',
    class_name VARCHAR(50) COMMENT '班级',
    major VARCHAR(100) COMMENT '专业',
    dormitory_id INT COMMENT '当前宿舍ID（NULL表示未入住）',
    check_in_date DATE COMMENT '最近一次入住日期',
    status VARCHAR(20) DEFAULT 'OUT' COMMENT '入住状态：IN（入住）/ OUT（退宿）',
    emergency_contact VARCHAR(50) COMMENT '紧急联系人',
    emergency_phone VARCHAR(20) COMMENT '紧急联系电话',
    create_time DATETIME DEFAULT NOW() COMMENT '创建时间',
    update_time DATETIME DEFAULT NOW() ON UPDATE NOW() COMMENT '更新时间',
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE,
    INDEX idx_student_no (student_no),
    INDEX idx_dormitory (dormitory_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学生信息表';


-- =====================================================
-- 3. 楼栋表
-- 功能覆盖：楼栋管理、分配楼管、数据隔离
-- =====================================================
CREATE TABLE building (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '楼栋ID',
    building_name VARCHAR(50) NOT NULL COMMENT '楼栋名称（如：A栋、致远楼）',
    floor_count INT DEFAULT 6 COMMENT '楼层总数',
    address VARCHAR(200) COMMENT '地址描述',
    building_admin_id INT COMMENT '楼栋管理员ID（关联user.id）',
    status TINYINT DEFAULT 1 COMMENT '状态：1正常 0停用',
    create_time DATETIME DEFAULT NOW() COMMENT '创建时间',
    update_time DATETIME DEFAULT NOW() ON UPDATE NOW() COMMENT '更新时间',
    FOREIGN KEY (building_admin_id) REFERENCES user(id) ON DELETE SET NULL,
    INDEX idx_admin (building_admin_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='楼栋表';


-- =====================================================
-- 4. 宿舍表
-- 功能覆盖：宿舍管理、入住/退宿/换宿人数联动、维修状态
-- =====================================================
CREATE TABLE dormitory (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '宿舍ID',
    building_id INT NOT NULL COMMENT '所属楼栋ID',
    room_number VARCHAR(20) NOT NULL COMMENT '房间号（如：301）',
    floor INT COMMENT '所在楼层（冗余字段，便于按楼层筛选）',
    capacity INT DEFAULT 4 NOT NULL COMMENT '总床位数量',
    current_count INT DEFAULT 0 NOT NULL COMMENT '当前已入住人数',
    status VARCHAR(20) DEFAULT 'AVAILABLE' COMMENT '状态：AVAILABLE（空闲）/ FULL（已满）/ REPAIR（维修中）',
    create_time DATETIME DEFAULT NOW() COMMENT '创建时间',
    update_time DATETIME DEFAULT NOW() ON UPDATE NOW() COMMENT '更新时间',
    UNIQUE KEY uk_building_room (building_id, room_number),
    FOREIGN KEY (building_id) REFERENCES building(id) ON DELETE CASCADE,
    INDEX idx_building (building_id),
    INDEX idx_status (status),
    INDEX idx_floor (floor)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='宿舍表';


-- =====================================================
-- 5. 入住/退宿/换宿流水表
-- 功能覆盖：入住管理、退宿管理、换宿管理、学生详情-记录查询
-- =====================================================
CREATE TABLE check_record (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '流水ID',
    student_id INT NOT NULL COMMENT '学生ID（关联student.id）',
    from_dormitory_id INT COMMENT '原宿舍ID（退宿/换宿时使用）',
    to_dormitory_id INT COMMENT '目标宿舍ID（入住/换宿时使用）',
    type VARCHAR(20) NOT NULL COMMENT '类型：IN（入住）/ OUT（退宿）/ CHANGE（换宿）',
    check_date DATETIME DEFAULT NOW() COMMENT '操作时间',
    operator_id INT COMMENT '操作人ID（关联user.id）',
    remark VARCHAR(500) COMMENT '备注说明',
    create_time DATETIME DEFAULT NOW() COMMENT '创建时间',
    FOREIGN KEY (student_id) REFERENCES student(id) ON DELETE CASCADE,
    FOREIGN KEY (from_dormitory_id) REFERENCES dormitory(id) ON DELETE SET NULL,
    FOREIGN KEY (to_dormitory_id) REFERENCES dormitory(id) ON DELETE SET NULL,
    FOREIGN KEY (operator_id) REFERENCES user(id) ON DELETE SET NULL,
    INDEX idx_student (student_id),
    INDEX idx_type (type),
    INDEX idx_date (check_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='入住/退宿/换宿流水表';


-- =====================================================
-- 6. 报修表
-- 功能覆盖：学生报修申请、楼管处理报修、状态流转
-- =====================================================
CREATE TABLE repair (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '报修ID',
    dormitory_id INT NOT NULL COMMENT '报修宿舍ID（关联dormitory.id）',
    reporter_id INT NOT NULL COMMENT '报修人ID（关联student.id）',
    repair_type VARCHAR(50) NOT NULL COMMENT '报修类型：水电/家具/门锁/网络/其他',
    description VARCHAR(500) COMMENT '问题描述',
    report_time DATETIME DEFAULT NOW() COMMENT '报修时间',
    status VARCHAR(20) DEFAULT 'PENDING' COMMENT '状态：PENDING（待处理）/ DOING（处理中）/ DONE（已完成）',
    handler_id INT COMMENT '处理人ID（关联user.id，通常为楼管）',
    handle_time DATETIME COMMENT '处理完成时间',
    handle_remark VARCHAR(500) COMMENT '处理备注（维修结果说明）',
    create_time DATETIME DEFAULT NOW() COMMENT '创建时间',
    update_time DATETIME DEFAULT NOW() ON UPDATE NOW() COMMENT '更新时间',
    FOREIGN KEY (dormitory_id) REFERENCES dormitory(id) ON DELETE CASCADE,
    FOREIGN KEY (reporter_id) REFERENCES student(id) ON DELETE CASCADE,
    FOREIGN KEY (handler_id) REFERENCES user(id) ON DELETE SET NULL,
    INDEX idx_dormitory (dormitory_id),
    INDEX idx_reporter (reporter_id),
    INDEX idx_status (status),
    INDEX idx_report_time (report_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='报修表';
```

### 三、核心字段设计说明

| 表               | 关键字段                                | 设计理由                              |
| ---------------- | --------------------------------------- | ------------------------------------- |
| **user**         | `role`                                  | 三个角色硬编码，Filter通过此字段拦截  |
| **user**         | `salt`                                  | 独立盐值，每个用户不同，防彩虹表攻击  |
| **student**      | `user_id`                               | 一对一关联，解耦账号与档案，支持扩展  |
| **student**      | `dormitory_id`                          | NULL表示未入住，便于统计空闲床位      |
| **building**     | `building_admin_id`                     | 楼管关联，数据隔离的核心字段          |
| **dormitory**    | `current_count`                         | 冗余字段，避免每次count(*)，提升性能  |
| **dormitory**    | `status=REPAIR`                         | 维修中状态，支持"统计维修数量"功能    |
| **check_record** | `from_dormitory_id` / `to_dormitory_id` | 同时记录原和目标，完美支持"换宿"溯源  |
| **repair**       | `status`                                | 三态流转，支持楼管"处理中→已完成"操作 |

### 六、功能→数据表映射清单（供开发参考）

| 功能模块                       | 读取/写入的表                    | 关键SQL说明                                                |
| ------------------------------ | -------------------------------- | ---------------------------------------------------------- |
| **登录**                       | user                             | `SELECT * FROM user WHERE username=?`                      |
| **修改密码**                   | user                             | `UPDATE user SET password=?, salt=? WHERE id=?`            |
| **用户管理（列表/搜索/CRUD）** | user                             | 分页 + `WHERE username LIKE ? OR real_name LIKE ?`         |
| **楼栋管理（CRUD）**           | building                         | 关联 `user` 查楼管姓名                                     |
| **宿舍管理（列表）**           | dormitory + building             | `LEFT JOIN building` 显示楼栋名称                          |
| **宿舍管理（筛选）**           | dormitory                        | `WHERE building_id = ?`                                    |
| **统计-总床位/入住率**         | dormitory                        | `SUM(capacity)`, `SUM(current_count)`                      |
| **统计-维修数量**              | dormitory                        | `SELECT COUNT(*) WHERE status='REPAIR'`                    |
| **学生管理（列表/搜索）**      | student + user + dormitory       | 三表联查显示完整信息                                       |
| **办理入住**                   | student, dormitory, check_record | 事务：更新student + 更新dormitory.current_count + 插入流水 |
| **办理退宿**                   | student, dormitory, check_record | 事务 + 检查 repair 表有无未完成                            |
| **办理换宿**                   | student, dormitory, check_record | 事务：旧宿舍-1，新宿舍+1，插入换宿流水                     |
| **我的宿舍（学生）**           | student + dormitory + user       | 查询同宿舍其他学生（排除自己）                             |
| **提交报修**                   | repair                           | `INSERT INTO repair`                                       |
| **报修列表（楼管）**           | repair + dormitory + building    | 只查 `building_id` 属于当前楼管的                          |
| **报修处理（状态流转）**       | repair                           | `UPDATE status` + 记录 handler_id 和 handle_time           |
| **数据统计（各楼栋入住率）**   | building + dormitory             | `GROUP BY building_id`                                     |
| **数据统计（最新5条报修）**    | repair + dormitory + building    | `ORDER BY report_time DESC LIMIT 5`                        |

### 七、数据隔离实现方案（楼管专用）

楼栋管理员登录后，所有查询必须带上**楼栋ID过滤**：

```sql
-- 楼管登录时：从Session获取 userId
-- 查询 building 表获取 building_admin_id = userId 的 building_id
-- 后续所有SQL示例：

-- 1. 查本楼栋宿舍
SELECT * FROM dormitory WHERE building_id = ?;

-- 2. 查本楼栋报修（通过 dormitory 关联）
SELECT r.* FROM repair r 
JOIN dormitory d ON r.dormitory_id = d.id 
WHERE d.building_id = ?;

-- 3. 查本楼栋学生（通过 dormitory 关联）
SELECT s.* FROM student s 
JOIN dormitory d ON s.dormitory_id = d.id 
WHERE d.building_id = ?;
```

### 八、建表执行顺序（重要）

由于存在外键约束，必须按以下顺序创建：

```
1. user        （无外键）
2. building    （外键 → user.id）
3. dormitory   （外键 → building.id）
4. student     （外键 → user.id, dormitory.id）
5. check_record（外键 → student.id, dormitory.id, user.id）
6. repair      （外键 → dormitory.id, student.id, user.id）
```

**删除/清空数据时，倒序执行。**
