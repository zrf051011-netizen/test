## 1. 项目概览

### 1.1 项目简介

本项目是一个**宿舍管理系统**，采用经典的 **JavaWeb 技术栈** 开发，实现了宿舍管理的核心业务功能，包括：

- **超级管理员**：用户管理、楼栋管理、宿舍管理、学生管理、数据统计
- **楼栋管理员**：宿舍管理、学生管理、报修管理、入住/退宿/换宿
- **学生**：个人信息、我的宿舍、报修申请、报修记录

### 1.2 技术选型思路

```
为什么选这些技术？
│
├── 初学者友好 ── Servlet + JSP 是JavaWeb的基础，易于理解
├── 轻量级 ───── 不使用Spring等重型框架，代码更纯净
├── 循序渐进 ─── 掌握这些后再学Spring Boot会更容易
└── 实用性强 ─── 这些技术在老系统中仍然广泛使用
```

## 2. 整体架构设计

### 2.1 B/S（Browser/Server 浏览器 / 服务器）

客户端只需要浏览器，程序、数据全放在服务端，升级只改服务器，跨平台。JavaWeb 全部基于此。

#### 完整 URL 全字段拆解（网络请求标准格式）

#### 标准模板

```Plain Text
https://api.example.com:8080/user/123/order?page=1&size=10#detail
```

逐段拆分说明：

1. **协议 protocol：https://**

   - 作用：定义客户端与服务器传输数据的规则

   - 常见值：`http://`、`https://`（加密）、`ws://`（websocket）、`ftp://`

   - 后缀`://`是协议固定分隔符

2. **域名 / 主机 host：**[**api\.example\.com**](https://api.example.com)

   - 作用：定位服务器地址，可域名 / IP

   - 分类：

     - 顶级域名：`example.com`

     - 子域名：`api`、`admin`、`www`

   - 等价写法：`120.92.xxx.xxx`（服务器 IP）

3. **端口 port：:8080**

   - 格式：域名后加冒号 \+ 数字

   - 默认省略规则：

     - http 默认 80 端口，可省略 `:80`

     - https 默认 443 端口，可省略 `:443`

   - 自定义服务必须显式写端口，如后端 8080、3000

4. **资源路径 path：/user/123/order**
   核心接口路径，分段由`/`分割，又称**路由路径**，分静态段、动态参数段

    - `/user`：静态资源模块（用户模块）

    - `/123`：路径参数（路径变量，代表用户 ID）

    - `/order`：子资源（用户订单）

    - 开头必须带`/`，代表根路径

5. **查询参数 query：?page=1\&size=10**

   - 起始标记：第一个`?`

   - 多个参数分隔：`&`

   - 单参数格式：`key=value`

   - 特点：可选、不影响资源唯一标识，多用于分页、筛选、搜索

6. **哈希锚点 hash：\#detail**

   - 起始标记：`#`

   - 特点：**不会发给后端服务器**，仅浏览器前端使用

   - 用途：页面内定位、前端 SPA 路由（Vue/React 路由）

#### HTTP/HTTPS（B/S 通信协议）

##### 1\. HTTP 基础

- 无状态协议：服务器不记住客户端，靠 Cookie/Session 保存状态

- 默认端口 80，HTTPS 加密端口 443

##### 2\. HTTP 请求组成

1. 请求行：请求方式 \+ URL \+ 协议版本（GET/POST 最常用）

2. 请求头：浏览器信息、Cookie、Content\-Type 等

3. 请求体：POST 参数、上传文件数据

##### 3\. 常用请求方式

- GET：参数拼在 URL，明文、大小有限、用于查询

- POST：参数在请求体，安全、可传大数据、提交表单 / 上传

- PUT/DELETE：RESTful 风格接口

##### 4\. HTTP 响应组成

1. 响应行：协议 \+ 状态码

2. 响应头：服务器信息、Cookie、数据类型

3. 响应体：HTML/JSON/ 图片

##### 5\. 常见状态码

- 200：成功；302：重定向；404：资源不存在；405：请求方式不允许；500：服务器代码异常

##### 6\. Cookie \& Session（解决 HTTP 无状态）

- Cookie：存在**浏览器**，键值对，可设置有效期，每次请求自动携带

- Session：存在**服务器**，每个用户独立会话，通过 Cookie 存 SessionID 关联用户登录状态

##### 7\. HTTPS

SSL 加密传输，防止数据抓包篡改，项目上线必备

## 3. 前端技术栈

### 3.1 JSP (Java Server Pages)

**什么是JSP？**
- 一种动态网页技术标准
- 在HTML中嵌入Java代码
- 服务器端执行，最终生成HTML返回给浏览器

**本项目中的用法：**
- 所有页面都是JSP文件
- 与Servlet配合，Servlet处理请求后转发到JSP渲染页面
- JSP负责展示数据，不处理业务逻辑

### 3.2 EL 表达式 (Expression Language)

**什么是EL表达式？**
- 用来简化JSP页面中数据的访问
- 语法：`${表达式}`
- 可以访问request、session、application等作用域中的数据

**本项目中的示例：**
```jsp
<!-- 访问用户信息 -->
<span>${realName}</span>

<!-- 访问分页数据 -->
共 ${pageBean.totalCount} 条记录

<!-- 访问对象属性 -->
${user.username}

<!-- 访问URL参数 -->
${param.keyword}
```

**EL表达式的工作原理（JavaBeans规范）：**
- `${user.username}` → 调用 `user.getUsername()`
- `${user.status == 1}` → 调用 `user.getStatus()` 后比较
- 属性名首字母大写 + get 前缀 = getter方法名

### 3.3 JSTL (JSP Standard Tag Library)

**什么是JSTL？**
- JSP标准标签库
- 用标签代替Java代码，让JSP更整洁
- 核心标签库：循环、判断、输出等

**本项目中的示例：**
```jsp
<!-- 引入JSTL -->
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

<!-- 循环遍历 -->
<c:forEach items="${pageBean.list}" var="user">
    <tr><td>${user.username}</td></tr>
</c:forEach>

<!-- 条件判断 -->
<c:if test="${empty list}">
    <td>暂无数据</td>
</c:if>

<!-- 多分支选择 -->
<c:choose>
    <c:when test="${user.role == 'ADMIN'}">超级管理员</c:when>
    <c:when test="${user.role == 'BUILDING_ADMIN'}">楼栋管理员</c:when>
    <c:otherwise>学生</c:otherwise>
</c:choose>
```

> 💡 **为什么用EL+JSTL而不是Java脚本？**
> - 代码更整洁，可读性好
> - 前后端分离更清晰
> - 不容易出错（比如忘记关闭标签）
> - 符合现代JSP开发规范

### 3.4 jQuery

**什么是jQuery？**
- 快速、简洁的JavaScript库
- 封装了DOM操作、事件处理、AJAX等常用功能
- 大大简化了JavaScript编程

**本项目中的用法：**
主要用于**AJAX异步请求**（修改密码功能）：

```javascript
$.ajax({
    url: '${pageContext.request.contextPath}/user/changePwd',
    type: 'POST',
    data: {
        oldPassword: oldPwd,
        newPassword: newPwd,
        confirmPassword: confirmPwd
    },
    dataType: 'json',
    success: function(result) {
        if (result.success) {
            alert('修改成功');
        } else {
            alert(result.message);
        }
    }
});
```

**AJAX的优势：**
- 不刷新整个页面，用户体验好
- 异步提交，不阻塞用户操作
- 只交换必要的数据，节省流量

### 3.5 CSS 样式

**什么是CSS？**
- 层叠样式表 (Cascading Style Sheets)
- 控制网页的外观和布局
- 与HTML内容分离，便于维护

**本项目中的样式特点：**
- 统一的侧边栏 + 顶部栏 + 内容区布局
- 表格、表单、按钮等通用组件样式
- 模态框（弹窗）样式
- 分页组件样式

### 4.2 Filter 过滤器

**什么是Filter？**
- 拦截请求，在Servlet执行之前/之后做一些处理
- 可以拦截多个资源，实现通用功能

**本项目中的 AuthFilter（权限过滤器）：**

```
请求 → AuthFilter → Servlet → 响应
         │
         ├─ 白名单检查（登录页、静态资源放行）
         ├─ 登录状态检查（未登录跳登录页）
         └─ 角色权限检查（无权限返回403）
```

**核心代码逻辑：**
```java
@WebFilter("/*")
public class AuthFilter implements Filter {
    public void doFilter(ServletRequest req, ServletResponse resp, FilterChain chain) {
        // 1. 白名单路径直接放行
        if (isWhiteList(uri)) { chain.doFilter(req, resp); return; }
        
        // 2. 检查登录状态
        User user = (User) session.getAttribute("loginUser");
        if (user == null) { response.sendRedirect(loginPage); return; }
        
        // 3. 检查角色权限
        if (!hasPermission(uri, user.getRole())) { 
            request.getRequestDispatcher("/403.jsp").forward(req, resp);
            return;
        }
        
        // 4. 放行
        chain.doFilter(req, resp);
    }
}
```

> 💡 **过滤器的典型应用场景：**
> - 权限控制
> - 字符编码统一设置
> - 日志记录
> - 敏感词过滤

## 5. 数据层技术

### 5.1 MySQL 数据库

**什么是MySQL？**

### 5.2 JDBC

**什么是JDBC？**
- Java Database Connectivity
- Java访问数据库的标准API
- 不同数据库有不同的驱动（Driver）

**JDBC的基本步骤：**
```
1. 加载驱动 → Class.forName("com.mysql.cj.jdbc.Driver")
2. 获取连接 → DriverManager.getConnection(url, user, password)
3. 创建语句对象 → PreparedStatement
4. 执行SQL → executeQuery() / executeUpdate()
5. 处理结果 → ResultSet
6. 关闭资源 → 关闭ResultSet、Statement、Connection
```

### 5.3 Apache Commons DBUtils

**什么是Commons DBUtils？**
- Apache提供的JDBC工具库
- 简化JDBC操作，减少样板代码
- 自动处理资源关闭
- 自动将ResultSet映射到JavaBean

**核心组件：**
- **QueryRunner**：执行SQL的核心类
- **ResultSetHandler**：结果集处理器
  - `BeanHandler`：结果映射为单个JavaBean
  - `BeanListHandler`：结果映射为JavaBean列表
  - `ScalarHandler`：结果映射为单个值

**本项目中的应用（BaseDao）：**
```java
// 查询单个对象
public T queryForBean(String sql, Class<T> clazz, Object... params) {
    return queryRunner.query(sql, new BeanHandler<>(clazz), params);
}

// 查询列表
public List<T> queryForList(String sql, Class<T> clazz, Object... params) {
    return queryRunner.query(sql, new BeanListHandler<>(clazz), params);
}

// 增删改
public int update(String sql, Object... params) {
    return queryRunner.update(sql, params);
}
```

> 💡 **为什么用DBUtils而不是原生JDBC？**
> - 不用手动关闭资源（不会忘记关连接导致泄漏）
> - 不用手动映射ResultSet到JavaBean（减少重复劳动）
> - 代码更简洁，专注于SQL本身

### 5.4 C3P0 数据库连接池

**什么是连接池？**
- 预先创建一些数据库连接，放在池中
- 需要时从池中取，用完归还
- 避免频繁创建/销毁连接的开销

```
没有连接池：
  请求1 → 创建连接 → 使用 → 销毁
  请求2 → 创建连接 → 使用 → 销毁
  （每次都要创建销毁，很慢）

有连接池：
  ┌─────────────────────┐
  │  连接池（已有5个连接） │
  └────────┬────────────┘
           ↓ 取一个
  请求1 → 使用 → 归还
  请求2 → 使用 → 归还
  （复用连接，性能好）
```

**C3P0的配置（c3p0-config.xml）：**
```xml
<c3p0-config>
    <default-config>
        <property name="driverClass">com.mysql.cj.jdbc.Driver</property>
        <property name="jdbcUrl">jdbc:mysql://localhost:3306/dormitory_system</property>
        <property name="user">root</property>
        <property name="password">123456</property>
        <property name="initialPoolSize">5</property>     <!-- 初始连接数 -->
        <property name="minPoolSize">5</property>         <!-- 最小连接数 -->
        <property name="maxPoolSize">20</property>        <!-- 最大连接数 -->
        <property name="maxIdleTime">600</property>       <!-- 最大空闲时间(秒) -->
    </default-config>
</c3p0-config>
```

**本项目中的DBUtil：**

## 6. 工具类与通用组件

### 6.1 MD5Util 密码加密工具

**为什么要加密密码？**
- 数据库中不存明文密码
- 即使数据库泄露，密码也不会直接暴露

**MD5 + 盐值加密：**
```
单纯MD5：
  123456 → e10adc3949ba59abbe56e057f20f883e
  （相同密码加密结果相同，容易被彩虹表破解）

MD5 + 盐值：
  123456 + abc123(盐) → MD5("123456abc123") → 不同的结果
  （每个用户的盐值不同，即使密码相同加密结果也不同）
```

**本项目中的实现：**
```java
public class MD5Util {
    // 生成随机盐值
    public static String generateSalt() { ... }
    
    // 加密：md5(密码 + 盐值)
    public static String encrypt(String password, String salt) {
        return DigestUtils.md5Hex(password + salt);
    }
}
```

**登录验证流程：**
```
1. 用户输入用户名和密码
2. 根据用户名查询用户，获取盐值和加密后的密码
3. 用输入的密码 + 盐值 加密
4. 比较加密结果与数据库中的密码是否一致
```

### 6.2 PageBean 分页工具

**分页的核心概念：**
- `page`：当前页码
- `size`：每页显示条数
- `totalCount`：总记录数
- `totalPage`：总页数 = ⌈totalCount / size⌉
- `offset`：偏移量 = (page - 1) * size

**SQL分页（MySQL）：**
```sql
SELECT * FROM user LIMIT 0, 5    -- 第1页，取前5条
SELECT * FROM user LIMIT 5, 5    -- 第2页，跳过5条取5条
SELECT * FROM user LIMIT 10, 5   -- 第3页，跳过10条取5条
```

**PageBean 设计：**
```java
public class PageBean<T> {
    private List<T> list;          // 当前页数据
    private int totalCount;        // 总记录数
    private int page;              // 当前页码
    private int size;              // 每页大小
    private int totalPage;         // 总页数
    
    // 计算属性
    public boolean getFirstPage() { return page == 1; }
    public boolean getLastPage() { return page == totalPage; }
    public boolean getHasPreviousPage() { return page > 1; }
    public boolean getHasNextPage() { return page < totalPage; }
}
```

**BaseDao中的分页查询：**
```java
public PageBean<T> queryPage(String sql, String countSql, Class<T> clazz, 
                              int page, int size, Object... params) {
    // 1. 查询总记录数
    int totalCount = queryForScalar(countSql, params);
    
    // 2. 查询当前页数据
    int offset = (page - 1) * size;
    String pageSql = sql + " LIMIT ?, ?";
    List<T> list = queryForList(pageSql, clazz, offset, size);
    
    // 3. 封装结果
    return new PageBean<>(list, totalCount, page, size);
}
```

### 6.3 BusinessException 业务异常

**什么是业务异常？**
- 业务逻辑中因为不满足条件而抛出的异常
- 比如：用户名已存在、密码不正确、宿舍已满等
- 区别于系统异常（数据库连接失败、空指针等）

**本项目中的用法：**
```java
// Service层抛出业务异常
public void deleteUser(int id) {
    User user = findById(id);
    if (user == null) {
        throw new BusinessException("用户不存在");
    }
    if ("ADMIN".equals(user.getRole())) {
        throw new BusinessException("不能删除超级管理员");
    }
    userDao.deleteById(id);
}

// Servlet层捕获并处理
try {
    userService.deleteUser(id);
    response.sendRedirect("/admin/user/list");
} catch (BusinessException e) {
    // 重定向到列表页，带错误信息
    response.sendRedirect("/admin/user/list?error=" + URLEncoder.encode(e.getMessage()));
}
```

> 💡 **为什么用自定义异常？**
> - 区分业务异常和系统异常
> - 统一处理业务错误信息展示
> - 代码更清晰，业务逻辑更明确

## 8. 安全机制

### 8.1 权限控制（RBAC）

**什么是RBAC？**
- Role-Based Access Control，基于角色的访问控制
- 用户 → 角色 → 权限

**本项目的角色划分：**

```
超级管理员 (ADMIN)
  ├── 用户管理（增删改查）
  ├── 楼栋管理（增删改查）
  ├── 宿舍管理（增删改查）
  ├── 学生管理（增删改查）
  └── 数据统计

楼栋管理员 (BUILDING_ADMIN)
  ├── 宿舍管理（所负责楼栋）
  ├── 学生管理（所负责楼栋）
  ├── 报修处理
  ├── 入住办理
  ├── 退宿办理
  └── 换宿办理

学生 (STUDENT)
  ├── 个人信息
  ├── 我的宿舍
  ├── 报修申请
  └── 报修记录
```

**权限校验位置：**
1. **AuthFilter**：URL级别的角色权限拦截
2. **Service层**：业务级别的数据范围校验（如只能看自己楼栋的数据）

### 8.2 密码安全

- MD5 + 盐值加密存储
- 每个用户有独立的随机盐值
- 修改密码需要验证原密码
- 新密码长度至少6位

### 8.3 登录状态管理

- 登录成功后用户信息存入 session
- AuthFilter 检查 session 中是否有登录用户
- 未登录自动跳转到登录页
- 退出登录时销毁 session

## 9. 核心功能实现示例

### 9.1 登录功能完整流程

```
用户输入用户名密码
    │
    ▼
LoginServlet.doPost()
    ├─ 获取用户名、密码
    ├─ 调用 userService.login(username, password)
    │    ├─ 根据用户名查询用户
    │    ├─ 用户不存在？→ 抛异常
    │    ├─ 密码不正确？→ 抛异常
    │    ├─ 账号被禁用？→ 抛异常
    │    └─ 返回用户对象
    ├─ 成功 → 存入session，跳转到对应首页
    └─ 失败 → 转发到登录页，显示错误
```

### 9.2 分页列表完整流程

```
用户访问 /admin/user/list?page=2
    │
    ▼
UserManageServlet.doGet() → list()
    ├─ 获取page参数，默认1
    ├─ 调用 userService.findByPage(2, 5, null)
    │    └─ 调用 userDao.findByPage(2, 5, null)
    │         ├─ SELECT COUNT(*) FROM user → totalCount
    │         └─ SELECT * FROM user LIMIT 5,5 → list
    ├─ PageBean 存入 request
    └─ 转发到 user_list.jsp
         ├─ <c:forEach> 遍历列表
         └─ 生成分页导航
```

### 9.3 添加学生（事务管理）

```
管理员提交新增学生表单
    │
    ▼
StudentManageServlet.doPost() → add()
    ├─ 封装 Student 对象
    ├─ 调用 studentService.addStudent(student, username, phone)
    │    ├─ 校验用户名已存在？
    │    ├─ 校验学号已存在？
    │    ├─ 创建用户（生成盐+加密密码）→ 返回userId
    │    ├─ 创建学生（关联userId）
    │    ├─ 学生创建失败？→ 删除用户，抛异常（回滚）
    │    └─ 成功 → 返回true
    ├─ 成功 → 重定向到列表页
    └─ 失败 → 重定向到列表页，带错误信息
```

> 💡 **为什么需要回滚？**
> 添加学生需要创建两条记录（用户 + 学生），如果学生创建失败，用户记录就成了"孤儿数据"，所以要回滚删除。
> （注：本项目使用手动回滚方式，更完善的做法是使用数据库事务）
